# Contributing

Contributions are welcome through issues and pull requests.

1. Discuss material public-schema or architecture changes in an issue first.
2. Install the supported Node.js version and run `npm ci`.
3. Make a focused change with tests and documentation.
4. Run `npm run check` and the relevant fixture suites.
5. Confirm no installed-game or third-party-mod content is included.

Security-sensitive reports belong in private vulnerability reporting, as described in `SECURITY.md`.

By contributing, you agree that your contribution is licensed under Apache-2.0.
